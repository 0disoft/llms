2024년 8월부터 2025년 8월 사이에 Deno에서 변경된 주요 기능들을 조사한 내용을 바탕으로, 지원이 중단되거나 제거된 기능과 새롭게 추가된 기능들을 정리해 드립니다.

---

### Deno 2.0 (2024년 10월)

**✔️ 새로 추가된 주요 기능**

* **Node.js 호환성 대폭 강화**:
    * **Node.js의 `process` 전역 객체**가 기본으로 제공됩니다.
    * `require(ESM)` 지원 등, Node.js API 호환성이 크게 개선되었습니다.
* **의존성 관리 개선**:
    * 새로운 잠금 파일 형식과 더 명확한 오류 메시지로 의존성 관리가 훨씬 편리해졌습니다.
* **안정화된 Web API**:
    * WebGPU, FFI (`Deno.dlopen()`), `Deno.createHttpClient()` 같은 기능이 `--unstable` 플래그 없이 사용할 수 있게 되었습니다.

**❌ 지원 중단/제거된 기능**

* **`window` 전역 객체**: 이 객체가 제거되고 Node.js의 `process` 객체로 대체되었습니다.
* **`deno bundle` 명령어**: 더 이상 지원하지 않습니다. (이후 Deno 2.4에서 `esbuild` 기반으로 복원됩니다)
* **`deno vendor` 명령어**: `deno.json` 파일의 `vendor` 옵션으로 대체되어 제거되었습니다.
* **사용 중단된 플래그**: `--unstable`, `--lock-write`, `--ts` 등의 플래그가 세부적인 플래그나 환경 변수로 대체되었습니다.

---

### Deno 2.2 (2025년 1월)

**✔️ 새로 추가된 주요 기능**

* **OpenTelemetry 내장 지원**:
    * 분산 추적(Distributed Tracing)을 위한 OpenTelemetry 기능이 내장되어 API 모니터링이 쉬워졌습니다. (초기에는 불안정 API로 제공)
* **린터 플러그인 API**:
    * 새로운 린터 플러그인 API를 통해 커스텀 린터 규칙을 추가할 수 있게 되었습니다.
* **`node:sqlite` 지원**:
    * Node.js의 `sqlite3`와 호환되는 내장 SQLite 모듈을 지원합니다.
* **`deno task` 개선**:
    * OS 신호 처리, 와일드카드 지원 등 `deno task`가 더욱 강력해졌습니다.

---

### Deno 2.3 (2025년 2월)

**✔️ 새로 추가된 주요 기능**

* **로컬 npm 패키지 지원**:
    * npm 패키지를 로컬 파일 시스템에서 직접 사용하고 수정하는 기능이 추가되었습니다.
* **`deno compile` 개선**:
    * FFI와 Node 네이티브 애드온 지원, 파일 제외 (`--exclude`) 등 단일 실행 파일 생성 기능이 향상되었습니다.
* **`deno fmt` 개선**:
    * HTML, CSS, JSON 등 내장 콘텐츠 포맷팅과 다양한 포맷터 옵션이 추가되었습니다.

---

### Deno 2.4 (2025년 7월)

**✔️ 새로 추가된 주요 기능**

* **`deno bundle` 복원**:
    * 이전에 제거되었던 `deno bundle` 명령어가 `esbuild`를 기반으로 다시 추가되었습니다.
* **안정화된 OpenTelemetry**:
    * Deno 2.2에서 추가된 OpenTelemetry 기능이 이제 안정적인 API로 제공됩니다.
* **새로운 CLI 기능**:
    * `deno run` 명령어에서 스크립트 커버리지 (`--coverage`)를 수집하는 기능이 추가되었습니다.
    * `deno update` 명령어로 의존성 업데이트가 더 간편해졌습니다.
* **더 강력해진 권한 제어**:
    * `--allow-net`에 와일드카드(`*`)와 CIDR 범위 사용을 허용하여 권한을 세밀하게 제어할 수 있게 되었습니다.
    * 새로운 `--deny-import` 플래그로 특정 모듈의 임포트를 차단할 수 있습니다.

**❌ 지원 중단/제거된 기능**

* **`patch` 옵션의 이름 변경**:
    * 로컬 npm 패키지에서 사용하던 `patch` 옵션의 이름이 `links`로 변경되었습니다. `patch`를 사용하면 경고가 표시됩니다.
