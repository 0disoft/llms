2024년 8월부터 2025년 8월 사이에 Flame 엔진에서 변경된 주요 기능들을 다음과 같이 정리했습니다. 이 기간 동안 여러 버전 업데이트가 있었으며, 주로 기존 기능 개선과 성능 최적화에 초점이 맞춰졌습니다.

***

### ➡️ 더 이상 지원하지 않는 기능 및 변경 사항

* **`HasGameRef` → `HasGameReference`로 변경:** 게임 인스턴스에 접근하는 데 사용했던 `HasGameRef` 믹스인이 `HasGameReference`로 이름이 바뀌고 더 안정적인 API를 제공하게 되었습니다. 기존 코드를 마이그레이션해야 합니다.
* **`Projector` 인터페이스 삭제:** 카메라 투영 관련 `Projector` 인터페이스가 삭제되고, `CameraComponent`가 이 역할을 직접 처리하도록 통합되었습니다.
* **`Vector2` 사용법 변경:** 특정 컴포넌트에서 `Vector2`가 `Offset`을 더 이상 직접 지원하지 않도록 변경되었습니다.
* **`query()` 반환 타입 변경:** 컴포넌트를 찾는 `query()` 메서드가 단일 컴포넌트를 반환할 경우, 더 이상 `List`가 아닌 `Component?`를 반환하도록 변경되었습니다. 이는 API 일관성을 위한 변경입니다.

***

### ➕ 새로 추가된 기능

* **포스트 프로세스 API (Post Process API):** 렌더링 파이프라인에 사용자 정의 셰이더를 쉽게 추가할 수 있는 API가 도입되었습니다. 이를 활용해 게임 화면에 독특한 시각 효과를 적용할 수 있습니다.
* **렌더 컨텍스트 API (Render Context API):** 렌더링 순서와 방식을 더 세밀하게 제어할 수 있게 되었습니다. 여러 컴포넌트를 묶어 한 번에 렌더링하는 등의 최적화가 가능합니다.
* **레이아웃 컴포넌트 (Layout Components):** UI 구성에 유용한 `Column`, `Row`와 같은 레이아웃 컴포넌트가 추가되어, 게임 내 UI를 보다 쉽게 만들 수 있습니다.
* **DevTools 통합:** 디버깅을 돕는 Flame DevTools가 개선되어, 게임 컴포넌트 구조와 렌더링 상태를 시각적으로 더 쉽게 파악할 수 있게 되었습니다.
* **새로운 이펙트 추가:** `SizeEffect.to` 등 다양한 효과들이 추가되어 컴포넌트 애니메이션을 더욱 풍부하게 만들 수 있습니다.
* **`SpawnComponent` 개선:** `SpawnComponent`에 새로운 기능들이 추가되어 게임 내에서 컴포넌트를 동적으로 생성하고 관리하는 기능이 강화되었습니다.
* **`ScrollTextBoxComponent` 추가:** 스크롤 가능한 텍스트 박스 컴포넌트가 추가되어, 대량의 텍스트를 게임 내 UI에 표시하기 쉬워졌습니다.
