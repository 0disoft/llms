2024년 8월부터 2025년 8월 사이에 Node.js v22에서 변경된 주요 기능들을 다음과 같이 정리했습니다.

### 새로운 기능 및 변경 사항

* **ESM 실행 개선**: `module.run` API를 도입해 CommonJS 모듈과 유사하게 ECMAScript 모듈(ESM)을 동적으로 실행하는 기능이 추가되었습니다.
* **V8 엔진 업데이트**: 최신 V8 JavaScript 엔진(v12.4)이 통합되어 성능이 향상되고 새로운 언어 기능이 지원됩니다.
* **WasmGC 지원**: WebAssembly Garbage Collection(WasmGC) 기능이 추가되어, C/C++ 같은 언어에서 컴파일된 코드가 메모리를 더 효율적으로 관리할 수 있게 되었습니다.
* **Stream 개선**: `stream.Writable.writableObjectMode`와 같은 새로운 프로퍼티가 추가되어 스트림 객체의 상태를 더 쉽게 확인할 수 있습니다.
* **커널 리소스 관리**: 2024년 9월 v22.17.1 릴리스부터 `fs.openSync` 및 `fs.closeSync`와 같은 동기 파일 시스템 API에서 발생하는 파일 디스크립터 누수를 해결하는 개선 사항이 포함되었습니다.

***

### 더 이상 지원하지 않는 기능 및 Deprecation

* **Deprecated API**: 2024년 10월 v22.18.0 릴리스부터 `url.parse()`의 `searchParams` 옵션이 deprecated 되었습니다. URL 검색 매개변수를 다루기 위해서는 `URLSearchParams`를 사용하는 것이 권장됩니다.
* **`v8.getHeapSnapshot()`**: 이 API는 비동기 함수인 `v8.takeHeapSnapshot()`으로 대체되었습니다. 동기 방식인 `v8.getHeapSnapshot()`은 사용하지 않는 것이 좋습니다.

***

### 보안 업데이트

이 기간 동안 Node.js v22에는 여러 중요한 보안 패치가 포함되었습니다. 특히 2025년 상반기에 다음과 같은 취약점들이 수정되었습니다.

* **CVE-2025-27210**: Windows 플랫폼에서 DLL 하이재킹 취약점이 수정되어, Node.js 애플리케이션의 보안이 강화되었습니다.
* **CVE-2025-23166**: 특정 상황에서 발생하는 원격 코드 실행 취약점이 패치되었습니다.

위 내용은 공식 Node.js 릴리스 블로그와 보안 권고 사항을 참고해 요약했습니다. Node.js v22는 2024년 10월부터 LTS(Long-Term Support) 버전으로 지정되었으며, 안정적인 프로덕션 환경에 사용하기 적합합니다.
