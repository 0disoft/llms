### Node.js v22: 2024년 8월 ~ 2025년 8월 주요 기능 변화 및 지원 종료 기능 분석 보고서

Node.js v22는 2024년 8월부터 2025년 8월까지의 기간 동안 중요한 전환기를 맞았습니다. 이 시기는 **'현재(Current)' 릴리스에서 '장기 지원(LTS)' 버전으로 승격**되는 기간으로, Node.js v22가 프로덕션 환경의 안정적인 기반으로 자리 잡게 되었습니다. 이 보고서는 이 기간 동안의 주요 기능 변화와 지원 종료 기능, 그리고 보안 취약점을 종합적으로 분석합니다.

---

### 1. Node.js v22의 신규 기능 및 주요 개선 사항

Node.js v22는 성능, 모듈 상호 운용성, 그리고 개발자 경험을 향상시키기 위한 다양한 기능을 도입하고 개선했습니다.

- **V8 JavaScript 엔진 업데이트**: V8 엔진이 **12.4 버전**으로 업데이트되었습니다.
  - **Maglev 컴파일러**: V8의 새로운 JIT 컴파일러가 기본적으로 활성화되어, 특히 단기 실행 CLI 애플리케이션의 성능을 크게 향상시켰습니다.
  - **새로운 API**: `WebAssembly 가비지 컬렉션`, `Array.fromAsync`, `Set 메서드` 및 이터레이터 헬퍼가 추가되었습니다.

- **ESM(ECMAScript 모듈) 지원 강화**:
  - **동기식 `require()` 지원**: `--experimental-require-module` 플래그를 통해 동기식 ES 모듈을 `require()`하는 실험적 지원이 추가되어, CommonJS와 ESM 간의 상호 운용성 격차를 줄였습니다.
  - **향상된 ESM 로더**: 모듈 로딩 동작을 더 정밀하게 사용자 정의할 수 있는 유연한 ESM 로더가 도입되었습니다.

- **실험적 기능 안정화**:
  - **`Fetch` API 안정화**: 웹 표준 `Fetch` API가 안정화되어 외부 라이브러리 없이도 일관된 HTTP 요청을 수행할 수 있게 되었습니다.
  - **안정적인 `Watch` 모드**: 파일 변경 시 애플리케이션을 자동으로 다시 시작하는 `Watch` 모드 명령줄 옵션(`--watch`)이 안정화되어 개발 효율성이 크게 향상되었습니다.

- **기타 개선 사항**:
  - **`node:fs` `glob` 및 `globSync`**: 파일 시스템 패턴 매칭을 위한 새로운 함수가 추가되어 파일 시스템 작업을 간소화했습니다.
  - **스트림 기본 High Water Mark 증가**: 스트림의 기본 버퍼 크기가 16KiB에서 64KiB로 증가하여 스트림 성능을 개선했습니다.
  - **`package.json` 스크립트 실행**: `node --run <script>` 플래그를 통해 `package.json` 스크립트를 직접 실행하는 실험적 기능이 추가되었습니다.

---

### 2. Node.js v22의 지원 종료 및 제거 기능

Node.js v22는 플랫폼을 현대화하고 간소화하기 위해 일부 레거시 API를 제거하고 지원 중단했습니다. 이러한 변경 사항은 이전 버전에서 마이그레이션하는 애플리케이션에 영향을 미칠 수 있습니다.

- **레거시 API 제거**: 일부 오래되고 덜 사용되는 API가 제거되었습니다. 해당 API에 의존하는 애플리케이션은 충돌을 피하기 위해 최신 대안으로 코드를 업데이트해야 합니다.
- **지원 중단된 기능**: 일부 기능이 지원 중단 대상으로 표시되어, 향후 버전에서 제거될 가능성이 높습니다. 개발자는 경고를 확인하고 제안된 대안으로 코드를 리팩토링하는 것이 권장됩니다.

### 3. Node.js v22의 주요 보안 업데이트

LTS 버전인 Node.js v22는 이 기간 동안 여러 중요한 보안 취약점 패치를 받았습니다. 이는 LTS 버전이라 할지라도 보안 위협에 대한 지속적인 관심이 필요함을 보여줍니다.

- **2025년 5월 14일 보안 릴리스**:
  - **CVE-2025-23166 (높음)**: 비동기 암호화 작업의 결함으로 원격 서비스 거부(DoS) 공격에 노출될 수 있는 취약점이 해결되었습니다.
  - **CVE-2025-23165 (낮음)**: 파일 시스템 메모리 누수 문제로 인해 시스템 메모리가 고갈될 수 있는 취약점이 수정되었습니다.

- **2025년 7월 15일 보안 릴리스**:
  - **CVE-2025-27210 (높음)**: Windows 장치 이름 경로 탐색 보호 우회 취약점이 해결되었습니다. 이는 Windows 기반 시스템의 무단 액세스를 방지하기 위한 중요한 패치입니다.

---

### 4. 결론 및 권장 사항

Node.js v22는 LTS 버전으로서의 안정성을 기반으로, 성능과 현대화를 위한 실질적인 발전을 보여주었습니다. 개발자와 조직은 다음과 같은 권장 사항을 고려하여 Node.js v22의 장점을 극대화해야 합니다.

- **최신 버전으로 업그레이드 우선순위 지정**: 성능 개선과 중요한 보안 패치를 활용하기 위해 Node.js v22.x, 특히 최신 패치 버전(예: v22.17.1 이상)으로의 업그레이드를 우선적으로 진행하세요.
- **포괄적인 마이그레이션 계획 수립**: 업그레이드 전 종속성 평가, 철저한 백업, 그리고 지원 중단된 API에 대한 코드 업데이트를 포함하는 상세한 계획을 세우세요.
- **보안 권고 지속적 모니터링**: LTS 버전이라도 새로운 보안 취약점이 발생할 수 있으므로, 공식 보안 권고를 지속적으로 모니터링하고 패치를 신속하게 적용하는 프로세스를 구축하세요.
- **새로운 기능 적극 활용**: `Watch` 모드, `Fetch` API 등 안정화된 기능을 활용하여 개발 효율성을 높이고, `Maglev 컴파일러`의 성능 이점을 활용하세요.
